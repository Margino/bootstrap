Paragraph

The HTML "p" element represents a paragraph of text. Paragraphs are usually represented in visual media as blocks of text that are separated from adjacent blocks by vertical blank space and/or first-line indentation. Paragraphs are block-level elements.

The start tag is required. The end tag may be omitted if the "p" element is immediately followed by an: address, article, aside, blockquote, div, dl, fieldset, footer, form, h1, h2, h3, h4, h5, h6, header, hr, menu, nav, ol, pre, section, table, ul, or another "p" element, or if there is no more content in the parent element and the parent element is not an "a" element.

Permitted ARIA roles: any.

Links

The HTML "a" element (or anchor element) creates a hyperlink to other web pages, files, locations within the same page, email addresses, or any other URL.

Tag omission: none, both the starting and ending tag are mandatory.

Note: When using target, consider adding rel="noopener noreferrer" to avoid exploitation of the window.opener API.

List, Highlight

The HTML "ul" element represents an unordered list of items, typically rendered as a bulleted list.

  • The HTML Strong Importance Element ("strong") indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.
  • The HTML Mark Text element ("mark") represents text which is marked or highlighted for reference or notation purposes, due to the marked passage's relevance or importance in the enclosing context.
  • The HTML "em" element marks text that has stress emphasis. The "em" element can be nested, with each level of nesting indicating a greater degree of emphasis.
  • The HTML "small" element makes the text font size one size smaller (for example, from large to medium, or from small to x-small) down to the browser's minimum font size. In HTML5, this element is repurposed to represent side-comments and small print, including copyright and legal text, independent of its styled presentation.

The HTML "ol" element represents an ordered list of items, typically rendered as a numbered list.

  1. The HTML Superscript element ("sup") specifies inline text1 which is to be displayed as superscript for solely typographical reasons. Superscripts are usually rendered with a raised baseline using smaller text.
  2. The HTML Abbreviation element ("abbr") represents an abbreviation or acronym; the optional title attribute can provide an expansion or description for the abbreviation. If present, title must contain this full description and nothing else.
  3. The HTML "hr" element represents a thematic break between paragraph-level elements (for example, a change of scene in a story, or a shift of topic with a section); historically, this has been presented as a horizontal rule or line. While it may still be displayed as a horizontal rule in visual browsers, this element is now defined in semantic terms, rather than presentational terms, so if you wish to draw a horizontal line, you should do so using appropriate CSS.

Blockquote

The HTML "blockquote" Element (or HTML Block Quotation Element) indicates that the enclosed text is an extended quotation. Usually, this is rendered visually by indentation (see Notes for how to change it). A URL for the source of the quotation may be given using the cite attribute, while a text representation of the source can be given using the "cite" element.
